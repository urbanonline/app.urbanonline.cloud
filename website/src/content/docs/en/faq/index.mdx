<details>
<summary>What technical requirements must the feeder meet in order to run the app?</summary>
The feeder model must be *Alma Pro*. Most *Alma Pro* feeders are compatible with the app, but the following *Alma Pro* feeders **cannot use** the app:

• Feeders equipped with a **first-generation touch terminal**.<br/>
• Feeders equipped with **parallel feeding using cups**.<br/>
• Feeders equipped with a **GSM module**.<br/>
</details>

<details>
<summary>What are the requirements for the router and the internet connection?</summary>

• The touchscreen terminal must be connected to an **Urban router**. All Urban routers, **even older models** are supported and can be used.<br/>
• The installed Urban router must be connected to the **internet** via the farm network. A **stable** internet connection is required all the times.<br/>
• The touchscreen terminal needs to establish a HTTPS-secured outbound connection to the Urban Cloud database server (via port: 443). No inbound traffic will occur, therefore, **no** dynamic DNS service (DDNS) and **no** VPN connection is required. The farm computer can communicate externally using any IP address (IPv4, IPv6), this address may also change.<br/>
</details>

<details>
<summary>Is the Urban Feed Monitor available for both Google (Android) and Apple (iOS) devices?</summary>
Yes, the app can be downloaded and installed from both the [Google Play Store]() (*Android*) and the [Apple Store]() (*iOS*)
</details>

<details>
<summary>Can I try the app for free before deciding on a subscription?</summary>
Yes, once you have [registered](#) for free within the app, you can load the machine and animal data from a demo farm and explore the app.
</details>

<details>
<summary>Can I still operate the touchscreen terminal remotely using a VNC client?</summary>
Yes, this option remains available. You can use the app and access the system via VNC at the same time.
</details>

<details>
<summary>Does the *Feed Monitor App* offer the functionality to back up and restore animal and machine data to rsp. from the cloud server?</summary>
No, this functionality is **not** available in the current version of the app.
</details>
